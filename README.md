Utility AI FrameworkAutor: Iker Garcia Repositorio: Utility-Al-Framework-Unity Versión de Unity Recomendada: 2022.3.62f3   1. Introducción a la IA UtilitariaLa Inteligencia Artificial Utilitaria es una arquitectura de toma de decisiones basada en un sistema de puntuación. A diferencia de las Máquinas de Estados Finitos o los Árboles de Comportamiento que siguen secuencias de ejecución rígidas, la IA Utilitaria evalúa en tiempo real un conjunto de acciones posibles y asigna una puntuación a cada una de ellas basándose en el estado interno del agente y su entorno.  El agente ejecuta la acción que obtiene la puntuación más alta en ese momento.  Una acción puede ponderar múltiples necesidades a la vez logrando que el comportamiento cambie de forma dinámica.  2. Para descargarlo y comenzar a usarlo en tu equipo, puedes optar por uno de los siguientes métodos:  Clonar mediante GitHub Desktop: En la página del repositorio, haz clic en el botón verde "Code" y selecciona "Open with GitHub Desktop". Descarga en ZIP: En la página del repositorio, haz clic en "Code", selecciona "Download ZIP" y extrae el contenido en tu equipo. Requisito de Versión: Es fundamental asegurarse de que el proyecto se ejecute bajo la versión 2022.3.62f3 o superior de Unity para asegurar la total compatibilidad y el correcto funcionamiento del NavMesh y las dependencias internas.  3. Componentes Principales del FrameworkBrainEs el componente base (MonoBehaviour) que se adjunta al GameObject del NPC. Almacena la lista de acciones disponibles para ese agente y es el encargado de ejecutar la evaluación matemática para determinar qué acción debe llevarse a cabo.  TickSchedulerEs un componente diseñado para la gestión del rendimiento. En lugar de que el Brain evalúe todas las acciones posibles de cada frame a través de Update el TickScheduler distribuye estas comprobaciones en intervalos de tiempo regulares. Para su funcionamiento, los agentes deben registrarse o enlazarse con él.  DynamicBlackboardActúa como un diccionario de memoria local o global de fácil acceso. El DynamicBlackboard permite almacenar y recuperar referencias como posiciones, actores cercanos o variables globales. Es una herramienta que se ofrece al usuario para que las Considerations y las Actions compartan información entre sí de forma limpia y desacoplada.  4. Implementación de Clases HijasConsideraciones (Consideration)Las Consideraciones son ScriptableObjects que responden a una pregunta concreta por ejemplo "¿Estoy cansado?", evaluando un dato. Para crear una nueva consideración en tu juego, debes heredar de la clase base Consideration.  GetRawInput (Imprescindible): Este es el único método que se tiene que sobrescribir obligatoriamente en tu consideración. En él, la lógica lee los sensores o el blackboard del agente y debe retornar un valor float normalmente normalizado entre 0.0 y 1.0. El framework recogerá automáticamente este valor crudo y lo evaluará contra la curva de respuesta configurada en el editor para obtener el puntaje final de utilidad.  Acciones (Action)Las Acciones también son ScriptableObjects y definen la secuencia lógica que ocurre cuando un comportamiento "gana" la evaluación del cerebro. Se crean heredando de la clase base Action.  OnExecute (Imprescindible): Este método se llama de manera continua siempre y cuando esta acción sea la elegida por el cerebro. Se debe sobrescribir para incluir el núcleo del comportamiento por ejemplo moverse hacia un objetivo, decrementar estadísticas, etc.  OnStart (Recomendable): Invocado una única vez cuando el agente comienza a realizar esta acción por primera vez tras ser escogida. Ideal para configurar parámetros, inicializar rutas del NavMesh o disparar animaciones iniciales.  OnStop (Recomendable): Invocado una única vez cuando la acción es interrumpida o se da por finalizada. Fundamental para limpiar referencias, reiniciar valores o detener animaciones, garantizando que el agente quede en un estado neutro antes de pasar a otra acción.  5. Resumen del Flujo de TrabajoInicialización: Añadir el componente Brain al NPC y disponer de un TickScheduler en escena que gestione las actualizaciones.  Crear Lógica: Programar las clases de tus consideraciones sobrescribiendo GetRawInput y de tus acciones sobrescribiendo OnExecute, y opcionalmente OnStart y OnStop.  Crear Assets: Generar los ScriptableObjects en el proyecto haciendo clic derecho sobre el menú correspondiente.  Configurar Curvas: Arrastrar tus consideraciones dentro de los arrays de tus Acciones y modificar las curvas matemáticas en el Inspector.  Asignación Final: Insertar estas acciones en la lista de comportamientos del Brain de tu agente.
